Aula 03: UML (Linguagem de Modelagem Unificada)

Duração: 2h  |  Modalidade: Online

1. Introdução à UML

A UML é uma linguagem padronizada para modelar sistemas orientados a objetos, permitindo representar visualmente requisitos, estrutura e comportamento.

  • Objetivos: padronizar notação para comunicação entre stakeholders, facilitar entendimento e documentação.
  • Versionamento: versões principais: UML 1.x (1997), UML 2.x (2005).
  • Diagramas: divididos em estruturais e comportamentais, cada um com propósito específico.

2. Diagramas Estruturais

Modelam a arquitetura estática do sistema, mostrando classes, interfaces e relacionamentos.

2.1 Diagrama de Casos de Uso

Descreve funcionalidades visíveis ao usuário (atores) sem detalhar implementação.

Elementos Principais

  • Atores: entidades externas que interagem com o sistema (usuário, sistema externo).
  • Casos de Uso: serviços ou funções oferecidos pelo sistema.
  • Relacionamentos: associação (ator–caso), inclusão (<>), extensão (<>).

Situação de Uso

Em projeto de e-commerce, casos de uso: "Fazer Pedido”, “Entregar Produto”, “Realizar Postagem”. Atores: Cliente, Funcionário, Transportador.

2.2 Diagrama de Classes

Mostra classes, atributos, operações e relacionamentos para representar estrutura de dados.

Elementos Principais

  • Classes: definidas em três seções: nome, atributos e métodos.
  • Relacionamentos: associação, agregação, composição, herança (generalização) e dependência.
  • Visibilidade: Pública (+), Privada (-), Protegida (#), Pacote (~).

Situação de Uso

Modelar domínio de vendas: classes Conta, Cliente, Vende. Relacionamentos: Cliente 1–* Endereço; Ciente 1–* Venda.

2.3 Diagrama de Componentes

Reúne partes substituíveis do sistema, como bibliotecas, módulos e pacotes.

Elementos Principais

  • Componentes: unidades de implementação (por exemplo, DLLs, JARs, serviços).
  • Interfaces: pontos de acesso público, exibidos como semicírculos ou retângulos com «interface».
  • Dependências: setas que indicam uso de interface por componente.

Situação de Uso

Em sistema distribuído, componentes: API Gateway, Serviço de Usuário, Banco de Dados; interfaces: REST, JDBC.

2.4 Diagrama de Implantação

Representa configuração de hardware e distribuição de componentes de software.

Elementos Principais

  • Nós: dispositivos ou máquinas físicas/virtuais (ex.: servidor, dispositivo móvel).
  • Artefatos: unidades de implantação, como arquivos executáveis ou bibliotecas.
  • Comunicações: conexões de rede entre nós (por exemplo, TCP/IP).

Situação de Uso

Descrever implantação em web: nó Dispositivo do Cliente, nó Servidor de Aplicação, nó Banco de Dados.

3. Diagramas Comportamentais

Modelam a lógica dinâmica e interação do sistema ao longo do tempo.

3.1 Diagrama de Sequência

Define colaboração entre objetos na execução de um fluxo de eventos, mostrando troca de mensagens ao longo do tempo.

Elementos Principais

  • Objetos/Participantes: apresentados no topo como retângulos, cada um com linha de vida.
  • Linhas de Vida: linhas pontilhadas verticais que representam a existência do objeto.
  • Mensagens: setas horizontais indicando chamadas de método ou retorno.
  • Ativação: retângulo fino sobre a linha de vida indicando execução de processo.

Situação de Uso

Fluxo de login: Cliente → LoginController: autenticar(); LoginController → UserService: validarCredenciais(); LoginController → Cliente: respostaAutenticada().

3.2 Diagrama de Atividade

Modela fluxo de trabalho (workflow) ou lógica de negócios, semelhante a um fluxograma.

Elementos Principais

  • Atividades: estados de ação, representados por retângulos com bordas arredondadas.
  • Nós de Decisão: losangos que direcionam fluxo condicional.
  • Nós de Início/Fim: círculos preenchidos (início) e círculos duplos (fim).
  • Fork/Join: barras que representam paralelismo e sincronização de fluxo.

Situação de Uso

Processo de pedido: Início → Verificar Estoque → [Disponível?] → Separar Item → Embalar → Fim ou Enviar Notificação de Falta e Fim.

3.3 Diagrama de Estado

Descreve estados pelos quais um objeto pode passar durante seu ciclo de vida, e transições entre estados.

Elementos Principais

  • Estados: representados por retângulos com cantos arredondados, mostrando nome do estado.
  • Transições: setas rotuladas com eventos que acionam mudança de estado.
  • Estado Inicial/Fim: círculo preenchido (inicial) e círculo duplo (final).
  • Ações/Atividades: podem ser associadas a transições ou estados, indicando comportamentos.

Situação de Uso

Ciclo de vida de uma tarefa: Novo → Em Progresso → Em Revisão → Concluído ou Cancelado.

4. Exemplos Práticos e Exercícios

4.1 Exemplo Completo: Biblioteca Online

Modelar sistema de empréstimo de livros:

  • Diagrama de Casos de Uso: Pesquisar Livro, Emprestar Livro, Devolver Livro.
  • Diagrama de Classes: Livro, Usuário, Empréstimo, Catálogo.
  • Diagrama de Sequência: fluxo de empréstimo, desde solicitante até confirmação.
  • Diagrama de Atividade: processo de devolução, incluindo checagem de multas e atualização de inventário.

4.2 Exercícios

  1. Crie um diagrama de Casos de Uso para um sistema de reservas de hospital, identificando atores (Paciente, Médico, Recepcionista) e casos de uso.
  2. Desenhe um diagrama de Classes para sistema de gerenciamento de curso, incluindo Aluno, Curso, Instrutor e relacionamentos adequados.
  3. Elabore um diagrama de Sequência para processo de pagamento online, incluindo Cliente, Gateway de Pagamento, Banco e Loja.
  4. Modele um diagrama de Atividade para fluxo de aprovação de folha de pagamento, desde envio até liberação de pagamento.
  5. Faça um diagrama de Estado para ciclo de vida de um incidente de TI, passando por Aberto, Em Investigação, Resolvido, Fechado.

5. Ferramentas UML

Existem diversas ferramentas para criar diagramas UML, desde soluções desktop até online.

Enterprise ArchitectDesktopModelagem completa, geração de código, gerenciamento de requisitos
Visual ParadigmDesktop/WebUML, BPMN, ERD, integra com IDEs e repositórios
PlantUMLTexto para DiagramaEspecificação em texto, integração com IDEs, geração automática
LucidchartWebColaboração em tempo real, templates, integração Google Drive
